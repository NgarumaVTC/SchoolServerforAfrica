#+Title: School Server Software Architecture (NixOS Edition)
#+Author: Axel Ramge
#+Date: 2026-05-05

* Router: MikroTik hAP ax3
- Firewall, Masquerading (NAT)
- DHCP server with pointer to iPXE/TFTP server (Boot process)
- Upstream DNS: Cloudflare Family (1.1.1.3 / 1.0.0.3) for zero-maintenance filtering of malware and adult content
- NTP master for the entire local network
- Isolated Wi-Fi for students
- Wireguard for remote management

* Server: NixOS Declarative Host (N100 -> später AMD 5950X)
** Infrastructure & Management (The "Nix Way")
- Single Source of Truth: Entire system state defined in `/etc/nixos` (Git-tracked, public on GitHub)
- NO Ansible, NO etckeeper required (NixOS natively handles declarative config management and rollbacks)
- Management Node (`mgnt1`): Central entry point for admin tools / rebuild scripts
- UPS integration via NUT (Network UPS Tools)
- Backup: "The African Way" – Weekly rotation of two rugged USB-C drives (Transcend StoreJet 25M3C)
  - ZFS Snapshots & ZFS Send/Receive to USB drives (blazing fast, incremental)

** Network & Core Services (NixOS Containers via systemd-nspawn)
- `web01` Container: 
  - Nginx web server
  - Delivery of iPXE boot images (HTTP)
  - Local documentation / School portal
- `ct-kiwix` Container (Offline Knowledge):
  - kiwix-serve for bandwidth-independent offline Wikipedia, Khan Academy, etc.
- `ct-auth` Container (Identity Provider):
  - LLDAP (Lightweight LDAP) for central user management (~120 students)
  - SQLite backend, minimal footprint, web-based UI
  - Provisional Master Password: "NgarumaVTC"

** Storage & User Data (Native ZFS & Bind-Mounts)
- Storage handled natively by the NixOS Host via ZFS (Datasets with quotas)
- NO NFS REQUIRED: Student homes are passed directly from the host into containers via blazing-fast native Bind-Mounts
- "The Teacher's Desk" (Das Lehrpult): Bind-Mounts for straightforward file exchange
  - `/tank/ClassMaterial` (Read-Only for students, Read-Write for teachers)
  - `/tank/DropBox` (Write-Only for students, Drop-In for assignments)

** Desktop Environments & Student Workspaces (NixOS Containers via systemd-nspawn)
- 20x NixOS containers (`ct90051` - `ct90070`) 
- Base Image managed via `studentssoftware.nix` (Firefox, LibreOffice, Zed, FreeCAD, GIMP, Python, etc.)
- GUI Stack: XFCE4 + xrdp (KISS-Principle, extremely low resource footprint)
- User environment dynamically linked upon login (Profiles)
- Students can safely install packages locally via `nix profile` without root privileges

* Thin Clients (20x Refurbished Business PCs / e.g. HP ProDesk or Dell OptiPlex Micro)
- Diskless, booting via iPXE
- Minimal RAM-based Linux OS (NixOS Netboot or Alpine)
- Kiosk mode: Direct start of `xfreerdp` (or `wlfreerdp`) in fullscreen, pointing to the student's assigned NixOS container
